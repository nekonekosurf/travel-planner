# 旅行リサーチスキル（travel-research）

## 目的
旅行先の各エリアについて、実際に旅行する人が「このサイトだけで行ける」レベルの情報を収集する。

## 調査手順

### Step 1: エリア概要の把握
1. 対象エリアの地理的位置を確認（Google Maps座標を取得）
2. 周辺の主要都市・交通ハブとの距離を確認
3. そのエリアの特徴（観光地/ローカル/ビーチ/山等）を把握

### Step 2: アクセス方法の調査
1. 前のエリアからの移動手段を**すべて**洗い出す
2. 各手段の所要時間・費用を調査
3. 具体的な手順を記載（「〇〇駅で△△行きの電車に乗り…」）
4. 予約方法（tiket.com, Traveloka, 現地購入等）を確認
5. 費用はIDRと日本円換算の両方を記載

### Step 3: 観光スポットの調査
1. 主要スポットを5-10個リストアップ
2. 各スポットについて以下を調査:
   - 入場料（外国人料金に注意）
   - 営業時間・定休日
   - ベスト訪問時間帯
   - 滞在目安時間
   - エリア中心部からのアクセス方法
   - 注意事項・Tips
3. 画像URLを収集（旅行ブログ、観光局サイト等から）
4. Google Maps URLを取得

### Step 4: 食事情報の調査
1. その地域の名物料理を調査
2. おすすめレストラン/ワルン/屋台を予算帯別に調査
   - 安い（〜30,000 IDR）
   - 中（30,000-80,000 IDR）
   - 高い（80,000 IDR〜）
3. 各店の人気メニュー・予算感を記載
4. 「これを食べろ」一品を選定
5. ローカルドリンク情報
6. 食の安全に関する注意事項

### Step 5: 宿泊情報の調査
1. エリアごとのおすすめ宿を予算帯別に調査
2. Booking.com, Agoda等での評価を参照
3. 予約方法のTipsを記載

### Step 6: 買い物・お土産の調査
1. その地域で有名なもの
2. 相場感
3. おすすめの店・マーケット

### Step 7: エリア実用情報
1. 徒歩圏内かどうか
2. ATM/両替所の位置
3. WiFi/SIM事情
4. 治安情報（特に夜間）
5. コンビニ・スーパー情報

## 品質基準
- 金額は必ずIDR + JPY換算を併記
- 「おすすめ」だけでなく「なぜおすすめか」を記載
- 曖昧な表現（「近い」「安い」等）は禁止。具体的な数値を使う
- Google Maps URLは必ず付与
- 画像は各スポットに最低1枚

## 出力形式
調査結果は `data/trip.json` の該当するDay構造に格納する。
スキーマは `docs/data-schema.md` を参照。
